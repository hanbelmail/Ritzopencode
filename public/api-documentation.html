Ritz Reservations API Documentation

Static reference for the Next.js App Router API endpoints used by staff tools, ticket automation, payment-proof uploads, and quote webhook forwarding.

Overview Authentication Tickets Payment Proof Quote Webhook Errors

Overview

Base URL

Use your deployed app origin, for example:

https://your-domain.com

Local development usually uses:

http://localhost:3000

Content Type

JSON endpoints accept and return application/json. Signed R2 upload URLs accept the image content type requested when the URL was created.

Data Source

Tickets are stored in Convex. API route handlers read and mutate ticket data through Convex server functions.

Authentication

The /api/tickets endpoints are protected by middleware. They can be accessed by an authenticated Convex Auth session or by a server-side API key.

MethodHeaderValue
Bearer tokenAuthorizationBearer $N8N_API_KEY
API key headerx-api-key$N8N_API_KEY
The N8N_API_KEY value lives in server environment variables. Do not expose it in browser code.

Ticket Endpoints

These endpoints are useful for staff tools and trusted automation. All examples below include the API key header.

PATCH /api/tickets/{id} Protected

Partially updates a ticket. If retailPrice, adjustment, checkIn, or checkOut changes, derived pricing is recalculated with current settings.

Request Body

FieldTypeDescription
statusstringTicket status such as PAYMENT VERIFIED or PAYMENT SUBMITTED.
retailPricenumber, null, or empty stringRetail room price. Empty string and null become null.
adjustmentnumber, null, or empty stringManual pricing adjustment. Empty string and null become null.
checkInstringDate in YYYY-MM-DD format.
checkOutstringDate in YYYY-MM-DD format.
paymentScreenshotKeystringPrivate R2 object key returned by the payment upload URL flow.

Example Request

curl -X PATCH "https://your-domain.com/api/tickets/8d4b6b3e-4a66-48b0-8a0d-fd8f28a07000" \
  -H "Authorization: Bearer $N8N_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "PAYMENT VERIFIED",
    "retailPrice": 2800,
    "adjustment": 0,
    "checkIn": "2026-08-10",
    "checkOut": "2026-08-14"
  }'

Example Response

{
  "ticket": {
    "id": "8d4b6b3e-4a66-48b0-8a0d-fd8f28a07000",
    "status": "PAYMENT VERIFIED",
    "retailPrice": 2800,
    "adjustment": 0,
    "nights": 4,
    "discountPct": 40,
    "total": 1680,
    "costPerNight": 420
  }
}

Payment Proof Endpoints

Payment proof images are stored in private Cloudflare R2 objects. The app creates short-lived signed URLs for upload and view access.

POST /api/payment-proof/upload-url JSON

Creates a 5-minute signed R2 URL that can upload one image file for an existing ticket.

Request Body

FieldRequiredDescription
ticketIdYesExisting ticket ID.
fileNameNoOriginal file name. Used only to infer the file extension.
contentTypeYesMust start with image/, for example image/png.

Example Request

curl -X POST "https://your-domain.com/api/payment-proof/upload-url" \
  -H "Content-Type: application/json" \
  -d '{
    "ticketId": "8d4b6b3e-4a66-48b0-8a0d-fd8f28a07000",
    "fileName": "payment.png",
    "contentType": "image/png"
  }'

Example Response

{
  "key": "payment-proofs/8d4b6b3e-4a66-48b0-8a0d-fd8f28a07000/4e8b0bb1-37a5-41a9-88fe-5d0423e8c000.png",
  "uploadUrl": "https://...signed-r2-url..."
}
PUT {uploadUrl} Cloudflare R2 signed URL

This is not an app route. Use the signed URL returned by /api/payment-proof/upload-url to upload the image bytes directly to R2.

Example Request

curl -X PUT "$UPLOAD_URL" \
  -H "Content-Type: image/png" \
  --data-binary "@payment.png"
After upload succeeds, save the returned key on the ticket with PATCH /api/tickets/{id} as paymentScreenshotKey.
POST /api/payment-proof/view-url JSON

Creates a 5-minute signed R2 URL for viewing a saved payment proof image.

Request Body

FieldRequiredDescription
ticketIdYesExisting ticket ID.
keyYesMust start with payment-proofs/{ticketId}/ and match the ticket's stored paymentScreenshotKey.

Example Request

curl -X POST "https://your-domain.com/api/payment-proof/view-url" \
  -H "Content-Type: application/json" \
  -d '{
    "ticketId": "8d4b6b3e-4a66-48b0-8a0d-fd8f28a07000",
    "key": "payment-proofs/8d4b6b3e-4a66-48b0-8a0d-fd8f28a07000/4e8b0bb1-37a5-41a9-88fe-5d0423e8c000.png"
  }'

Example Response

{
  "viewUrl": "https://...signed-r2-view-url..."
}

Quote Webhook Endpoint

POST /api/quote-webhook JSON

Forwards a quote ticket object to the configured external webhook URL when webhook sending is enabled in settings.

Request Body

FieldRequiredDescription
ticketYesJSON object to send to the configured webhook URL.

Example Request

curl -X POST "https://your-domain.com/api/quote-webhook" \
  -H "Content-Type: application/json" \
  -d '{
    "ticket": {
      "id": "8d4b6b3e-4a66-48b0-8a0d-fd8f28a07000",
      "guests": ["Ava Taylor"],
      "email": "guest@example.com",
      "phone": "+1 555 000 0000",
      "checkIn": "2026-08-10",
      "checkOut": "2026-08-14",
      "roomType": "Deluxe Ocean View",
      "status": "QUOTE REQUESTED"
    }
  }'

Example Responses

{ "sent": true, "status": 200 }
{ "skipped": true, "reason": "Webhook is disabled" }
If webhook sending is enabled but no webhook URL is configured, this endpoint returns 400.

Error Format

API errors return JSON with an error string.

{ "error": "Ticket not found" }
StatusCommon Reason
400Invalid JSON body, missing required fields, invalid image content type, invalid payment proof key, webhook URL not configured.
401Missing Convex Auth session and missing or invalid N8N_API_KEY for protected ticket endpoints.
404Ticket not found or payment proof not found for ticket.
500Server configuration or persistence failure.
502Configured quote webhook returned a non-success status.
Source routes: app/api/tickets, app/api/payment-proof, and app/api/quote-webhook. Convex Auth HTTP routes are managed by Convex Auth and are not custom app API endpoints.
